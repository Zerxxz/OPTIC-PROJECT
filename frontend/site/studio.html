How it works

  1. Prompt → JSON. OPTIC's LLM (via OpenRouter) returns a canonical StrategySpec validated against a Zod schema. The model can refuse, but it cannot lie about the schema.
  2. JSON → sha256. The spec is sorted-keyed and serialized deterministically. The sha256 is the on-chain identity of the strategy.
  3. JSON → Walrus. The full spec is uploaded to Walrus as a blob; the blob ID is returned and linked from the on-chain StrategyRef.
  4. Commit on-chain. A core::update_strategy_hash PTB anchors the new hash + blob ID on the Agent object. From that moment on, the Move contract refuses to act on any strategy whose hash does not match the committed one.
  5. No LLM on the hot path. The orchestrator reads the spec from Walrus (or a local cache) and runs it deterministically. LLM is only used to translate intent.

Why this matters

🛡️ Trustless AI

The "AI" part is auditable: anyone can read the spec from Walrus, re-run it, and verify the hash matches what's on-chain. No trust required.

🪪 Verifiable lineage

Every StrategyHashChanged event is emitted on-chain with old/new hashes. The full history of "what the agent was doing" is queryable from chain data alone.

🧪 Backtestable

Because the spec is deterministic, you can re-run it against historical DeepBook fills and reproduce the exact PnL curve — perfect for the Backtest Harness.
