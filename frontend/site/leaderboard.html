How is this computed?

  1. Read AgentRegistry — a singleton shared object lists all created agents.
  2. For each agent, read the core::Agent object (name, owner, status, strategy hash).
  3. For each agent, read the linked core::PnL object via dynamic fields (realized PnL, trade count, volume).
  4. Sort by the chosen metric (default: Sharpe = realized / volume * √365).
  5. Render. The browser just visualizes — anyone can re-derive this on their own server.

SDK module: @optic/sdk > computeLeaderboard(opts). The Walrus Site itself doesn't store anything — the index.html, JS, and leaderboard data are all fetched from Walrus / Sui RPC at view time.
